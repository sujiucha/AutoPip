# AutoPip Installer Pro

用于 MCDReforged 的全自动插件 Python 依赖包管理与安装工具。

> **Important**
> 本插件旨在彻底解决 MCDR 新手服主在安装复杂插件时遇到的“缺少依赖”、“环境污染”以及“pip 指令报错”等环境配置痛点。
> 建议在安装任何带有 `requirements.txt` 的新插件后使用本工具。

---

## 功能介绍

**核心机制**
- [x] 强制绑定当前运行 MCDR 的宿主 Python 环境进行安装，绝不污染系统全局环境
- [x] 采用独立后台线程执行网络 IO 与编译任务，避免阻塞主线程导致服务器卡顿或 Watchdog 警告
- [x] 捕获底层子进程流，将 pip 安装进度与报错实时重定向至 MCDR 后台控制台

**智能扫描**
- [x] 自动扫描 `plugins` 目录下所有的 `requirements.txt` 依赖清单
- [x] 自动跳过 `.disabled` 禁用插件目录以及 `__pycache__` 缓存目录
- [x] 支持自定义最大扫描深度，避免冗余遍历降低服务器性能

**交互体验**
- [x] 游戏内提供基于 RText 构筑的交互式界面，支持鼠标悬停提示与点击快捷执行
- [x] 任务锁机制，防止多名管理员同时发起安装导致文件死锁
- [x] 内置国内 PyPI 镜像源支持，显著降低依赖下载失败率

## 配置项

插件首次加载后，会自动在 `config/autopip.json` 生成配置文件。

| 配置项 | 配置说明 | 类型 | 示例 (默认值) |
| :--- | :--- | :--- | :--- |
| `pip_mirror` | Python 依赖下载镜像源。若留空 `""` 则使用官方默认源 | string | `https://pypi.tuna.tsinghua.edu.cn/simple` |
| `max_scan_depth` | 依赖扫描的最大目录层级深度。建议保持默认以节约性能 | integer | `2` |

> **Note**
> 修改配置文件后，无需重启整个服务器，可直接通过 `!!pip reload` 指令进行热重载。

## 指令预览

> 所有指令均需要权限等级 3（管理员/控制台）才能执行。

**常规指令**

```text
!!pip check    -> 扫描并列出所有缺少的依赖项（提供游戏内点击执行按钮）
!!pip install  -> 在后台创建进程，一键安全安装所有扫描到的依赖
!!pip list     -> 获取当前 Python 环境已安装的第三方库列表及版本号
